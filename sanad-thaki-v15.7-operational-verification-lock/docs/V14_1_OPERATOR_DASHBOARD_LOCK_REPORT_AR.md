# تقرير المرحلة — v14.1.0 Commercial Operator Dashboard Lock

## القرار
هذه نسخة تجارية أحدث مبنية على `v14.0.0-commercial-full-ui-candidate`، وتضيف متطلبات أدمن سند ذكي المعتمدة مع الحفاظ على خفة المنتج والهوية التجارية.

## ما تم إنجازه
- إضافة دور مستقل: `SANAD_ADMIN` لأدمن سند ذكي / مشغل المنصة.
- إضافة داشبورد خفيف لأدمن سند ذكي.
- إدارة شركات العملاء: إضافة شركة، تفعيل، إيقاف، حالة الشركة، الباقة، وحدود الاستخدام.
- متابعة استخدام الفواتير ورسائل واتساب على مستوى إجمالي لكل شركة فقط.
- إدارة تذاكر الدعم الفني من جهة مشغل المنصة.
- إضافة سجلات تدقيق خاصة بعمليات أدمن سند ذكي.
- منع أدمن سند ذكي من دخول دورة العميل المالية: الفواتير، الاعتماد، المطابقة، الإرسال المالي.
- إبقاء الواجهة خفيفة وبلا تضخم.

## أثر الأمان
- الصلاحيات مفروضة من Backend وليست إخفاء أزرار فقط.
- أدمن سند ذكي لا يملك صلاحيات `INVOICE_CREATE` أو `INVOICE_APPROVE` أو `BANK_MANAGE` أو `MATCH_APPROVE` أو `WHATSAPP_SEND_APPROVED`.
- إضافة عزل RLS مع نطاق خاص لمشغل المنصة للعمليات الإدارية والملخصات فقط.
- إضافة تدقيق `platform_audit_logs` لكل عمليات المنصة.
- استمرار منع `companyId` القادم من العميل في العمليات الحساسة.

## ما لم يتم السماح به
- لا عرض لتفاصيل فواتير العملاء في لوحة تشغيل المنصة.
- لا حذف فواتير العملاء.
- لا اعتماد مالي.
- لا مطابقة بنك.
- لا إرسال واتساب مالي.
- لا تعديل بيانات مالية داخل شركات العملاء.

## الفحوصات
- تحديث فحص الإصدار ليتأكد من وجود ملفات ومؤشرات لوحة أدمن سند ذكي.
- استمرار فحص عدم كشف PostgreSQL على منفذ عام.
- استمرار فحص وجود RLS و Backend permissions.

## المتبقي قبل بيانات العملاء الحقيقية
- نشر نظيف على Staging.
- اختبار قبول سريع: دخول أدمن سند ذكي، إضافة شركة، إضافة مستخدم أول، دخول مستخدم الشركة، دورة فاتورة واحدة، عزل بين شركتين.
- فحص أمني خارجي.
- اختبار نسخ احتياطي واستعادة.
- تفعيل HTTPS وأسرار إنتاجية حقيقية.

## قرار القبول
النسخة جاهزة للرفع على GitHub وStaging كـ Commercial Release Candidate أحدث، وليست لبيانات عملاء حقيقية قبل بوابات الأمان والنسخ الاحتياطي والعزل.
