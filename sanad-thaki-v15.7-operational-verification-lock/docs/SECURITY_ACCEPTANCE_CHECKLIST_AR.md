# قائمة قبول الأمان قبل بيانات العملاء

لا نعتمد بيانات عملاء حقيقية حتى تنجح هذه البنود:

- [ ] /health يعمل على AWS Staging.
- [ ] JWT secrets قوية ومغيرة.
- [ ] ALLOW_DEMO_LOGIN=false.
- [ ] PostgreSQL غير مكشوف للإنترنت.
- [ ] companyId لا يقبل من العميل.
- [ ] RLS مفعل ومجرب.
- [ ] شركة A لا ترى شركة B.
- [ ] واتساب لا يعمل إلا بعد APPROVED.
- [ ] Admin لا يرى تقارير مالية.
- [ ] Accountant لا يعتمد ماليًا.
- [ ] Finance Manager لا يدير مستخدمين.
- [ ] Backup يعمل.
- [ ] Restore يعمل.
- [ ] HTTPS يعمل.
- [ ] فحص خارجي مكتمل.
