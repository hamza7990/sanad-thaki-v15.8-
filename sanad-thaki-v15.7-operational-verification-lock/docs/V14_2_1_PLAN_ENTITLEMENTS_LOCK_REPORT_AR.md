# تقرير v14.2.1 — قفل ميزات الباقات

## القرار
تم تثبيت صياغة الباقات النهائية في واجهة البرنامج وصفحة الهبوط والسياسات، مع فرض الميزات من الخلفية وليس من الواجهة فقط.

## الصياغة المعتمدة
- الأساسية 99: تنظيم فواتير العملاء ومراجعتها.
- النمو 249: تسريع التحصيل ومتابعة العملاء وإرسال واتساب ومطابقة بنك.
- الاحترافية 499: للفواتير الأعلى التي تحتاج تحكمًا ماليًا أوسع وتقارير متقدمة.

## القفل الخلفي
- Basic: لا واتساب، لا مطابقة بنك، لا تقارير متقدمة، لا تصدير متقدم، لا دعم أولوية.
- Growth: واتساب + مطابقة بنك + تقارير تحصيل + تصدير + حدود متوسطة.
- Professional: استخدام أعلى + رسائل أكثر + تقارير متقدمة + دعم أولوية.

## الأمان
تمت إضافة حراس Backend/API لرفض الميزات غير المشمولة في الباقة حتى لو حاول المستخدم استدعاء API مباشر.

## التنبيه
هذه نسخة مرشحة للرفع على Staging والفحص الأمني، وليست مخصصة لبيانات عملاء حقيقية قبل الفحص الأمني الخارجي واختبار العزل والنسخ الاحتياطي.
