# دليل قبول Staging قبل أول عميل — سند ذكي

## 1) الإعداد

- انسخ `.env.production.example` إلى `.env.production`.
- عبئ القيم الحقيقية: قاعدة التحكم، Redis، Secrets Manager، الدومين، JWT، SMTP، و`METRICS_BEARER_TOKEN`.
- لا تستخدم مفاتيح إنتاج Meta أو بيانات عملاء حقيقية في Staging.

## 2) التشغيل

```bash
docker compose -f docker-compose.production.yml up -d --build
```

## 3) الفحص الآلي

```bash
export SANAD_BASE_URL=https://staging.your-domain.example
export METRICS_BEARER_TOKEN=...
bash RUN-LIVE-PRODUCTION-ACCEPTANCE-LINUX.sh
```

## 4) اختبار العزل اليدوي الإلزامي

1. ادخل بحساب منصة.
2. أنشئ شركة A وأدمنها.
3. أنشئ شركة B وأدمنها.
4. ادخل إلى A وأنشئ فاتورة.
5. ادخل إلى B وتأكد أن الفاتورة غير ظاهرة.
6. جرّب تزوير `companyId` في body أو query إن وجد؛ يجب أن يرفض النظام أو يتجاهله.

## 5) اختبار دورة الفاتورة

- رفع/قراءة فاتورة.
- تصحيح يدوي.
- إرسال للمراجعة.
- اعتماد المدير المالي.
- ظهور واتساب فقط بعد الاعتماد وبحسب الباقة.

## 6) اختبار البنك

- ارفع CSV فيه مبلغ عشري مثل `1234.56`.
- تأكد أنه لا يتحول إلى `123456`.
- نفذ المطابقة واعتمدها.
- حاول اعتماد مطابقة شقيقة؛ يجب أن ترجع 409 أو تُرفض.

## 7) بوابة القبول

لا تنتقل للإنتاج إلا إذا نجحت جميع البنود أعلاه وسُجلت النتيجة في تقرير داخلي بتاريخ واسم المسؤول.
