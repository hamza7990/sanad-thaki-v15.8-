# تقرير v14.2.7 — OCR 95 + مجموعة الفواتير + ربط محاسبي آمن

النطاق المنفذ محدود على ثلاثة بنود مع الحفاظ على قلب سند ذكي:

1. تحسين قراءة الفواتير بهدف تشغيلي 95%:
   - إضافة حد قبول تشغيلي 0.95.
   - أي قراءة أقل من 95% أو ناقصة الحقول تتحول للمراجعة اليدوية.
   - تحسين صور الفواتير في المتصفح قبل الإرسال: تباين/إضاءة/رمادي لتقوية القراءة.
   - عدم اعتماد أو قفل أي فاتورة من OCR وحده.

2. مجموعة الفواتير للمحاسب:
   - المحاسب يستطيع رفع عدة فواتير من صفحة الفواتير.
   - تظهر كل فاتورة في جدول مراجعة مع الحقول والثقة والحالة.
   - لا تحفظ المجموعة إلا بعد اكتمال كل الحقول المطلوبة.

3. ربط محاسبي آمن:
   - إضافة جدول مفاتيح ربط Integration API Keys مع company_id و RLS.
   - إضافة endpoint لاستيراد فواتير من نظام محاسبي خارجي عبر مفتاح شركة معزول.
   - المفتاح لا يعتمد على companyId من العميل، بل يحدد الشركة من hash المفتاح في السيرفر.

ملاحظات مهمة:
- 95% هنا معيار قبول وتشغيل، وليس وعدًا مطلقًا لكل صورة رديئة أو فاتورة ناقصة.
- الحماية الأساسية ما زالت: Company Isolation + RBAC/SOD + Audit Logs.
- النسخة لا تستخدم لبيانات عملاء حقيقية قبل التحقق الأمني الخارجي.
