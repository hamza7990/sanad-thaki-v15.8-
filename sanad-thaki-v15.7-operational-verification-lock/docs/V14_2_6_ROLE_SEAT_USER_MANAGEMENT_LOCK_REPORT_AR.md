# تقرير v14.2.6 — قفل إدارة المستخدمين وحدود الأدوار

هذه نسخة تثبيت قبل العملاء، وليست توسعة ميزات. هدفها إصلاح إدارة المستخدمين التي ظهرت أثناء اختبار أدمن الشركة.

## الإصلاحات

- أصبح أدمن الشركة يضيف الموظف بالاسم والبريد والدور فقط.
- تم منع أدمن الشركة من تعيين كلمة مرور دائمة للموظف.
- النظام يولد كلمة مرور مؤقتة تلقائيًا.
- كلمة المرور المؤقتة محفوظة كـ hash فقط داخل قاعدة البيانات.
- صلاحية الدعوة/كلمة المرور المؤقتة 24 ساعة.
- أول دخول للموظف يجبره على تغيير كلمة المرور قبل استخدام النظام.
- في Staging فقط تظهر كلمة المرور المؤقتة لأدمن الشركة مرة واحدة للاختبار.
- في Production لا يتم عرض كلمة المرور المؤقتة.
- أضيفت إعادة إرسال دعوة/إعادة تعيين كلمة مرور.
- أضيفت أرشفة المستخدم بدل الحذف النهائي.
- المستخدم المؤرشف لا يستطيع الدخول ولا يُحسب ضمن حدود الباقة.
- تبقى سجلات المستخدم المؤرشف محفوظة في Audit Log.
- تم إصلاح حدود الباقات حسب الدور وليس العدد الإجمالي.

## حدود الباقات المعتمدة

- الأساسية: أدمن شركة واحد + محاسب واحد + مدير مالي واحد.
- النمو: أدمن شركة واحد + محاسبان + مديران ماليان.
- الاحترافية: أدمن شركة واحد + 4 محاسبين + 4 مدراء ماليين.

## الحماية

- لا يتم قبول companyId من العميل.
- الشركة تؤخذ من JWT/session فقط.
- كل عمليات إدارة المستخدمين تمر عبر withTenant وAudit Log.
- لم يتم تغيير قلب الفواتير أو البنك أو الواتساب أو عزل الشركات.
