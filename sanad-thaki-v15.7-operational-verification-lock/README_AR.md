# سند ذكي v14.2.4-core-guard-lock — Core Protection Lock

هذه نسخة إقفال لحماية قلب البرنامج من Regression. لا تضيف ميزات جديدة، بل تضيف حارسًا برمجيًا يمنع سقوط مسارات الفواتير، التصحيح، المراجعة، الاعتماد، واتساب بعد الاعتماد، البنك، المطابقة، العزل، الصلاحيات، الباقات، الدعم، والنسخ الاحتياطي.

الأمر الأهم قبل أي اعتماد:

```bash
node scripts/core-regression-guard.mjs
```

أو تشغيل الفحص الكامل:

```bash
./RUN-FINAL-SMOKE-TEST-LINUX.sh
```

على ويندوز:

```cmd
RUN-FINAL-SMOKE-TEST-WINDOWS.cmd
```

هذه النسخة مخصصة لـ Production-Grade Staging فقط، وليست لاستخدام بيانات عملاء حقيقية قبل الأمن السيبراني الخارجي، واختبار عزل الشركات، والنسخ الاحتياطي/الاستعادة.

# سند ذكي v14.2.4-core-guard-lock — Staging Repair Lock

هذه النسخة تعتمد v14.2.2 وتصلح أخطاء تشغيل مهمة قبل السيرفر: ترتيب SQL، تطبيق المايغريشن تلقائيًا، إعادة رفع/قراءة الفاتورة من الواجهة إلى API، ضبط حدود المستخدمين، وتنظيف واجهة مشغل المنصة.

- الأساسية 99: تنظيم فواتير العملاء ومراجعتها.
- النمو 249: تسريع التحصيل ومتابعة العملاء وإرسال واتساب ومطابقة بنك.
- الاحترافية 499: للفواتير الأعلى التي تحتاج تحكمًا ماليًا أوسع وتقارير متقدمة.

باقة الأساسية لا تشمل واتساب أو مطابقة البنك أو التقارير المتقدمة أو التصدير المتقدم أو دعم الأولوية. يتم فرض ذلك من Backend/API وقاعدة البيانات وليس من الواجهة فقط.

## التشغيل على Staging

بعد رفع النسخة إلى الخادم وتشغيل Docker Compose، افتح:

```text
http://SERVER-IP/login
```

إذا كانت قاعدة البيانات جديدة سيظهر معالج التهيئة الأولية لإنشاء حساب مدير النظام الأول. بعد ذلك تستخدم نفس البريد وكلمة المرور للدخول.

## المسار التشغيلي

المحاسب يدخل الفاتورة ثم يضغط `تثبيت/إرسال للمراجعة`. بعد ذلك لا يحق له تعديل الفاتورة. المدير المالي يدخل على `مراجعة الفواتير` ويعتمد ماليًا. بعد الاعتماد يظهر للمحاسب زر `إرسال` في واتساب العملاء. المدير المالي يضيف عمليات البنك ويشغل المطابقة ثم يعتمد أو يرفض المطابقة. التقارير تعرض ملخص الفواتير والتحصيل.

## تنبيه أمني

هذه النسخة مرشحة للفحص الأمني والتشغيل على Staging فقط. لا تُستخدم مع بيانات عملاء حقيقية قبل إكمال فحص الأمن السيبراني الخارجي، والتحقق من العزل بين الشركات، والنسخ الاحتياطي والاستعادة.
